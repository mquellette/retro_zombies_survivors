# Asset Requirements — Vampire Survivors Style (Telegram Game)

## Visual Style
- **Pixel art**, вайб 80-х, прямой референс: *Zombies Ate My Neighbors* (SEGA/SNES)
- Палитра: насыщенная, контрастная, 16-bit эстетика
- Ориентация: **Portrait** (мобильный Telegram)

---

## 1. Разрешение и сетка

| Параметр | Значение |
|---|---|
| Базовое разрешение игры | **360×640 px** (логическое, скейлится на устройство) |
| Размер тайла | **16×16 px** (базовый юнит сетки) |
| Размер персонажа (спрайт) | **16×16 px** или **16×32 px** (если нужен рост) |
| Размер врагов | **16×16 px** (мелкие), **32×32 px** (средние), **48×48 px** или **64×64 px** (боссы) |
| Размер снарядов/эффектов | **8×8 px** или **16×16 px** |
| UI-элементы | Кратно 8 px |

> **Важно для Replicate:** генерировать ассеты в **4x разрешении** (т.е. персонаж 16×16 → генерить как 64×64), затем даунскейлить nearest-neighbor. Это даёт AI больше пикселей для работы и сохраняет чёткость.

---

## 2. Спрайт-шиты персонажей

### Формат генерации
- Генерировать **каждый кадр отдельно** (Replicate лучше работает с одиночными изображениями)
- Прозрачный фон (или однотонный фон для последующего удаления: **#FF00FF** magenta)
- Вид: **top-down с лёгким наклоном** (3/4 view), как в ZAMN

### Анимации игрока (минимум для PoC)
| Анимация | Кадры | Описание |
|---|---|---|
| Idle | 2 | Лёгкое покачивание |
| Walk down | 4 | Движение вниз (основное) |
| Walk up | 4 | Движение вверх |
| Walk side | 4 | Движение вбок (зеркалим для другой стороны) |

### Анимации врагов (минимум для PoC)
| Анимация | Кадры | Описание |
|---|---|---|
| Walk | 2-4 | Шагающий зомби/монстр |
| Death | 3-4 | Гибель (пиксельный взрыв/распад) |

### Промпт-шаблон для Replicate (персонажи)
```
pixel art character sprite, 16-bit style, top-down 3/4 view,
[ОПИСАНИЕ ПЕРСОНАЖА], single frame, clean pixels,
no anti-aliasing, limited color palette,
style of Zombies Ate My Neighbors SNES,
transparent background, 64x64 pixels
```

## 3. Тайлы окружения

### Формат
- **16×16 px** (генерить как **64×64 px**, даунскейл)
- Бесшовные (tileable) где возможно
- Без прозрачности для ground-тайлов

### Набор для PoC (1 уровень — «Кладбище»)
| Тайл | Варианты |
|---|---|
| Трава (земля) | 2-3 варианта для разнообразия |
| Дорожка/тропинка | Прямая, поворот, перекрёсток |
| Надгробия | 3-4 варианта (декор/препятствие) |
| Забор | Прямой, угол, столб |
| Дерево | 1-2 варианта (блокирующий объект) |

### Промпт-шаблон для Replicate (тайлы)
```
pixel art tile, 16-bit style, top-down view,
[ОПИСАНИЕ ТАЙЛА], seamless tileable texture,
no anti-aliasing, limited color palette,
style of Zombies Ate My Neighbors SNES,
64x64 pixels, dark moody atmosphere
```

## 5. UI-ассеты

> UI собирается в **Figma**, затем экспортируется.

### Целевое разрешение экрана
- **Дизайн в Figma:** 360×640 (1x) или 720×1280 (2x)
- Экспорт: **PNG @1x, @2x, @3x**

### Элементы для PoC
| Элемент | Описание |
|---|---|
| Кнопка «Играть» | Пиксельная рамка, 80s шрифт |
| Экран выбора уровня | Карточка уровня с превью |
| HUD (в игре) | HP-бар, таймер, счётчик убийств, уровень/XP |
| Джойстик (виртуальный) | Полупрозрачный, пиксельный стиль |
| Экран паузы | Оверлей с кнопками |
| Экран Game Over | Статистика, кнопка «Заново» |

### Шрифт
- Использовать пиксельный шрифт (например: **Press Start 2P**, **Silkscreen**, **Pixel Operator**)
- Если нужен кириллический — **Press Start 2P** поддерживает кириллицу

## 6. Требования к экспорту

| Параметр | Значение |
|---|---|
| Формат | **PNG** (спрайты, тайлы), **WebP** (UI, если размер критичен) |
| Прозрачность | PNG с альфа-каналом для спрайтов |
| Сжатие | Без lossy-сжатия для пиксель-арта |
| Масштабирование | Только **nearest-neighbor** (никакого bilinear/bicubic!) |
| Именование | `category_name_frame.png` → `player_walk_down_01.png` |

---

## 7. Палитра (рекомендация)

Для единообразия стиля — ограничить палитру. Подходящие варианты:
- **ENDESGA-32** (32 цвета, отлично подходит под 16-bit)
- **SWEETIE-16** (16 цветов, более минималистично)
- Кастомная палитра на основе скриншотов ZAMN

> При генерации в Replicate указывать `limited color palette` в промпте, затем при необходимости ремапить на выбранную палитру постобработкой.
